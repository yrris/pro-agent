## 2. 控制变量

- 模型：`deepseek-v4-pro`，temperature=0；SDK 隐式 retry=0，显式 retry 等待=[1.0, 2.0] 秒。
- 每个任务先由真实 planner 冻结一次计划，serial/parallel 回放完全相同的 steps、`<sep>` 分支和依赖 DAG；正式执行禁止动态改写冻结计划。
- 并发度：[1, 2, 4, 8]；每个 task/config 预热 1 次、正式 3 次；预热 80 条不进入指标，正式 240 条。

## 10. 实验局限

- planner 只在冻结阶段真实调用一次；正式 latency 衡量冻结 DAG 从进入生产图到 summary 的编排端到端耗时，不包含面向用户请求时的实时规划延迟。
- temperature=0 仍不能消除远端模型、网络、服务端排队与磁盘 prompt cache 的随机性；轮换顺序与三次重复只能降低而不能消除该影响。
- 数据集覆盖 calculator 与虚构固定知识库检索，不代表写文件、生图、外部 MCP、网页抓取或有副作用工具的性能。
- token 来自 provider usage；若未返回 cache-read 明细，成本按全部 input cache miss 计算，属于保守上界而非账单金额。
- 本实验隔离 Send 分支并发收益，动态 replan 机制由生产回归测试覆盖，但不评价其规划质量。
